1. Overview & How It Works

The Saved Farms feature lets a logged-in buyer bookmark farms they are interested in. They can add farms from the listings page and view or remove them on a dedicated Saved Farms page.

User Flow

  1. Buyer browses Farm Listings (/buyer/farmlistings).
  2. Each farm card has a "Save Farm" button. Clicking it saves that farm to the buyer's personal list and the button changes to "Saved" (disabled).
  3. Buyer clicks "Go To My Saved Farms" in the right sidebar.
  4. The Saved Farms page (/buyer/saved_farms) loads all saved farms for the logged-in buyer.
  5. Each saved farm has a "Remove" button that instantly removes it from the list.
Nothing is brand new here architecture-wise. The backend already had skeleton code for this feature — the key work is fixing broken relationships, correcting broken endpoints, and building the frontend UI.

2. Why We Changed the Database Relationship

This is the most important conceptual fix in this implementation. Understanding it will help you not repeat the same mistake elsewhere.

The Original (Broken) Code

The original Buyer.java had:

@OneToMany()
@JoinColumn(name="buyer_id", referencedColumnName="id")
private List<Seller> savedFarms;

@OneToMany with @JoinColumn works by adding a foreign key column to the other table. In this case, it adds a buyer_id column directly to the sellers table:

-- What the old code creates:
sellers table:
  id | shop_name | description | buyer_id   <-- WRONG

This means each seller row can only reference one buyer. The moment a second buyer tries to save the same farm, they would overwrite the first buyer's reference or the database would reject it entirely.

The Correct Approach — ManyToMany

Think about the relationship from both sides:

  • One buyer can save many farms
  • One farm can be saved by many buyers

That is a many-to-many relationship. The correct solution is a separate join table that only records which buyer saved which seller:

-- What the new code creates:
saved_farms table:
  buyer_id | seller_id
  ─────────────────────
  1        | 3          <-- buyer 1 saved farm 3
  2        | 3          <-- buyer 2 also saved farm 3  ✓
  1        | 7          <-- buyer 1 also saved farm 7

The sellers table stays completely clean — no foreign key pointing to buyers. Any number of buyers can independently save the same farm.

Step 1 — Buyer.java Backend

Path: backend/src/main/java/com/prym/backend/model/Buyer.java

What to change

  • Replace the @OneToMany + @JoinColumn block with a @ManyToMany + @JoinTable block.
  • Initialize the list to new ArrayList<>() so it is never null.
  • Add an unsaveFarm() method.
  • Add import java.util.ArrayList; at the top.
Buyer.java — full file
package com.prym.backend.model;

import jakarta.persistence.*;
import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;
import java.util.ArrayList;
import java.util.List;

@Entity
@Table(name = "buyers")
@Data
@NoArgsConstructor
@AllArgsConstructor
public class Buyer {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @OneToOne
    @JoinColumn(name = "user_id", nullable = false, unique = true)
    private User user;

    private String preferredCuts;

    @ManyToMany
    @JoinTable(
        name = "saved_farms",
        joinColumns = @JoinColumn(name = "buyer_id"),
        inverseJoinColumns = @JoinColumn(name = "seller_id")
    )
    private List<Seller> savedFarms = new ArrayList<>();

    public void saveFarm(Seller farm) {
        savedFarms.add(farm);
    }

    public void unsaveFarm(Seller farm) {
        savedFarms.remove(farm);
    }
}
Why new ArrayList<>()? Without it, savedFarms is null for any buyer that was created before this feature was built. Calling savedFarms.add() on a null list throws a NullPointerException. Initializing it to an empty list is safe in all cases.

Step 2 — BuyerService.java Backend

Path: backend/src/main/java/com/prym/backend/service/BuyerService.java

What to change

  • Add SellerRepository as a dependency (inject it via the constructor).
  • Change saveFarm to accept a Long sellerId instead of a Seller object — the service is responsible for looking up the seller from the database.
  • Add a duplicate-save guard (don't add the same farm twice).
  • Add a new unsaveFarm method.
  • Add @Transactional to both saveFarm and unsaveFarm.
Why does the service look up the Seller by ID? The controller receives raw HTTP input from the frontend. It should only extract the sellerId number and pass it to the service. The service is the right place to do database lookups and business logic — this keeps the controller thin and testable.
BuyerService.java — full file
package com.prym.backend.service;

import com.prym.backend.model.Buyer;
import com.prym.backend.model.Seller;
import com.prym.backend.model.User;
import com.prym.backend.repository.BuyerRepository;
import com.prym.backend.repository.SellerRepository;
import com.prym.backend.repository.UserRepository;
import org.springframework.transaction.annotation.Transactional;
import org.springframework.stereotype.Service;
import java.util.List;

@Service
public class BuyerService {

    private final BuyerRepository buyerRepository;
    private final UserRepository userRepository;
    private final SellerRepository sellerRepository;

    public BuyerService(BuyerRepository buyerRepository,
                        UserRepository userRepository,
                        SellerRepository sellerRepository) {
        this.buyerRepository = buyerRepository;
        this.userRepository = userRepository;
        this.sellerRepository = sellerRepository;
    }

    public Buyer createBuyerProfile(Long userId, String preferredCuts) {
        User user = userRepository.findById(userId)
                .orElseThrow(() -> new RuntimeException("User not found"));
        if (buyerRepository.existsByUserId(userId)) {
            throw new RuntimeException("Buyer profile already exists");
        }
        Buyer buyer = new Buyer();
        buyer.setUser(user);
        buyer.setPreferredCuts(preferredCuts);
        return buyerRepository.save(buyer);
    }

    public Buyer getBuyerProfile(Long userId) {
        return buyerRepository.findByUserId(userId)
                .orElseThrow(() -> new RuntimeException("Buyer profile not found"));
    }

    @Transactional
    public Buyer updateBuyerProfile(Long userId, String preferredCuts, String phoneNumber) {
        Buyer buyer = buyerRepository.findByUserId(userId)
                .orElseThrow(() -> new RuntimeException("Buyer profile not found"));
        buyer.setPreferredCuts(preferredCuts);
        if (phoneNumber != null && !phoneNumber.isBlank()) {
            User user = buyer.getUser();
            user.setPhoneNumber(phoneNumber);
            userRepository.save(user);
        }
        return buyerRepository.save(buyer);
    }

    public List<Seller> getSavedFarms(Long userId) {
        Buyer buyer = buyerRepository.findByUserId(userId)
                .orElseThrow(() -> new RuntimeException("Buyer profile not found"));
        return buyer.getSavedFarms();
    }

    @Transactional
    public Buyer saveFarm(Long userId, Long sellerId) {
        Buyer buyer = buyerRepository.findByUserId(userId)
                .orElseThrow(() -> new RuntimeException("Buyer profile not found"));
        Seller seller = sellerRepository.findById(sellerId)
                .orElseThrow(() -> new RuntimeException("Seller not found"));
        // Guard: don't save the same farm twice
        if (!buyer.getSavedFarms().contains(seller)) {
            buyer.saveFarm(seller);
        }
        return buyerRepository.save(buyer);
    }

    @Transactional
    public Buyer unsaveFarm(Long userId, Long sellerId) {
        Buyer buyer = buyerRepository.findByUserId(userId)
                .orElseThrow(() -> new RuntimeException("Buyer profile not found"));
        Seller seller = sellerRepository.findById(sellerId)
                .orElseThrow(() -> new RuntimeException("Seller not found"));
        buyer.unsaveFarm(seller);
        return buyerRepository.save(buyer);
    }
}

Step 3 — BuyerController.java Backend

Path: backend/src/main/java/com/prym/backend/controller/BuyerController.java

What to change

  • Fix the saveFarm method: add @RequestBody annotation and change the parameter from a Seller object to a Map<String, Object> so we extract just the sellerId.
  • Add a new DELETE /api/buyer/all/{sellerId} endpoint that calls unsaveFarm.
  • Remove the now-unused import com.prym.backend.model.Seller; line from the imports.
Why was the old saveFarm broken? It was declared as saveFarm(Seller farm) with no @RequestBody annotation. Without @RequestBody, Spring does not deserialize the JSON request body into the parameter — the farm object would always be null/empty. Every save attempt would silently fail or throw a NullPointerException.
BuyerController.java — full file
package com.prym.backend.controller;

import com.prym.backend.model.Buyer;
import com.prym.backend.model.User;
import com.prym.backend.model.Seller;
import com.prym.backend.service.BuyerService;
import com.prym.backend.repository.UserRepository;
import org.springframework.http.ResponseEntity;
import org.springframework.security.core.context.SecurityContextHolder;
import org.springframework.web.bind.annotation.*;

import java.util.List;
import java.util.Map;

@RestController
@RequestMapping("/api/buyer")
public class BuyerController {

    private final BuyerService buyerService;
    private final UserRepository userRepository;

    public BuyerController(BuyerService buyerService, UserRepository userRepository) {
        this.buyerService = buyerService;
        this.userRepository = userRepository;
    }

    private Long getLoggedInUserId() {
        String email = SecurityContextHolder.getContext().getAuthentication().getName();
        User user = userRepository.findByEmail(email)
                .orElseThrow(() -> new RuntimeException("User not found"));
        return user.getId();
    }

    // POST /api/buyer/profile — create buyer profile
    @PostMapping("/profile")
    public ResponseEntity<?> createProfile(@RequestBody Map<String, String> request) {
        try {
            Long userId = Long.parseLong(request.get("userId"));
            if (!getLoggedInUserId().equals(userId))
                return ResponseEntity.status(403).body(Map.of("error", "You can only create your own profile"));
            Buyer buyer = buyerService.createBuyerProfile(userId, request.get("preferredCuts"));
            return ResponseEntity.ok(buyer);
        } catch (RuntimeException e) {
            return ResponseEntity.badRequest().body(Map.of("error", e.getMessage()));
        }
    }

    // GET /api/buyer/profile/{userId} — get buyer profile
    @GetMapping("/profile/{userId}")
    public ResponseEntity<?> getProfile(@PathVariable Long userId) {
        try {
            if (!getLoggedInUserId().equals(userId))
                return ResponseEntity.status(403).body(Map.of("error", "You can only view your own profile"));
            return ResponseEntity.ok(buyerService.getBuyerProfile(userId));
        } catch (RuntimeException e) {
            return ResponseEntity.badRequest().body(Map.of("error", e.getMessage()));
        }
    }

    // PATCH /api/buyer/profile/{userId} — update buyer profile
    @PatchMapping("/profile/{userId}")
    public ResponseEntity<?> updateProfile(@PathVariable Long userId,
                                            @RequestBody Map<String, String> request) {
        try {
            if (!getLoggedInUserId().equals(userId))
                return ResponseEntity.status(403).body(Map.of("error", "You can only edit your own profile"));
            Buyer buyer = buyerService.updateBuyerProfile(
                    userId, request.get("preferredCuts"), request.get("phoneNumber"));
            return ResponseEntity.ok(buyer);
        } catch (RuntimeException e) {
            return ResponseEntity.badRequest().body(Map.of("error", e.getMessage()));
        }
    }

    // GET /api/buyer/all — return saved farms for logged-in buyer
    @GetMapping("/all")
    public ResponseEntity<?> getSavedFarms() {
        try {
            List<Seller> savedFarms = buyerService.getSavedFarms(getLoggedInUserId());
            return ResponseEntity.ok(savedFarms);
        } catch (RuntimeException e) {
            return ResponseEntity.status(500).body(Map.of("error", e.getMessage()));
        }
    }

    // PATCH /api/buyer/all — save a farm (body: { sellerId })
    @PatchMapping("/all")
    public ResponseEntity<?> saveFarm(@RequestBody Map<String, Object> body) {
        try {
            Long sellerId = Long.parseLong(body.get("sellerId").toString());
            Buyer buyer = buyerService.saveFarm(getLoggedInUserId(), sellerId);
            return ResponseEntity.ok(buyer);
        } catch (RuntimeException e) {
            return ResponseEntity.status(500).body(Map.of("error", e.getMessage()));
        }
    }

    // DELETE /api/buyer/all/{sellerId} — remove a farm from saved list
    @DeleteMapping("/all/{sellerId}")
    public ResponseEntity<?> unsaveFarm(@PathVariable Long sellerId) {
        try {
            Buyer buyer = buyerService.unsaveFarm(getLoggedInUserId(), sellerId);
            return ResponseEntity.ok(buyer);
        } catch (RuntimeException e) {
            return ResponseEntity.status(500).body(Map.of("error", e.getMessage()));
        }
    }
}

Step 4 — farm.js Frontend

Path: uiserver/src/api/farm.js

What to change

  • Fix getSavedFarms(): the URL was /api/buyer/ (wrong). Change it to /api/buyer/all and add credentials: 'include' so the session cookie is sent.
  • Fix saveFarm(): change the parameter from a full farm object to just sellerId, and update the URL from the placeholder "URL" to /api/buyer/all.
  • Add a new unsaveFarm(sellerId) function that calls DELETE /api/buyer/all/{sellerId}.
What is credentials: 'include'? The backend uses session-based authentication stored in a cookie. By default, fetch() does not send cookies on cross-origin or same-origin requests unless you explicitly tell it to. Without this, the backend receives an unauthenticated request and returns a 403 error.
farm.js — full file
export async function getAllFarms() {
    const response = await fetch('/api/seller/all');
    return response.json();
}

export async function getFarm(id) {
    const farms = await getAllFarms();
    return farms.find((f) => f.id === Number(id));
}

export async function getSavedFarms() {
    const response = await fetch('/api/buyer/all', {
        credentials: 'include',
    });
    return response.json();
}

export async function saveFarm(sellerId) {
    const response = await fetch('/api/buyer/all', {
        method: 'PATCH',
        headers: { 'Content-Type': 'application/json' },
        credentials: 'include',
        body: JSON.stringify({ sellerId }),
    });
    return response.json();
}

export async function unsaveFarm(sellerId) {
    const response = await fetch(`/api/buyer/all/${sellerId}`, {
        method: 'DELETE',
        credentials: 'include',
    });
    return response.json();
}

Step 5 — FarmListingsPage.jsx Frontend

What to change

  • Add a savedIds state (a Set of seller IDs the buyer has already saved) — loaded on mount alongside the farm list.
  • Add a handleSave function that calls the API and updates the local state.

Why use a Set for savedIds? Sets have O(1) lookup time — savedIds.has(farm.id) is instant regardless of how many farms are saved. An array would require looping through every element each render.
Important: The Save button sits inside a <Link> tag (which navigates to the farm detail page). Clicking the Save button would normally also trigger the Link navigation. The e.preventDefault() call inside handleSave stops this from happening.

  ├─ saveFarm(farmId)

  │
  └─ setSavedIds(prev => new Set([...prev, farmId]))

  ├─ BuyerService.getSavedFarms()
  │    └─ look up Buyer by userId
  │    └─ return buyer.getSavedFarms()
  │         └─ JPA loads all Sellers from saved_farms join table for this buyer
  │
  └─ setSavedFarms(data)
       └─ renders farm cards

Removing a Saved Farm

Buyer clicks "Remove" on a saved farm card
  │
  ├─ handleUnsave(farm.id) called
  │
  ├─ unsaveFarm(sellerId)
  │    └─ DELETE /api/buyer/all/{sellerId}
  │
  ├─ BuyerController.unsaveFarm()
  │    └─ calls BuyerService.unsaveFarm(loggedInUserId, sellerId)
  │
  ├─ BuyerService.unsaveFarm()
  │    └─ look up Buyer by userId
  │    └─ look up Seller by sellerId
  │    └─ buyer.unsaveFarm(seller) → savedFarms.remove(seller)
  │    └─ buyerRepository.save(buyer)
  │         └─ deletes row from saved_farms table
  │
  └─ setSavedFarms(prev => prev.filter(f => f.id !== sellerId))
       └─ card disappears instantly from UI

8. API Reference

Method URL Auth Required Body Description
GET /api/buyer/all BUYER session — Returns list of saved farms for the logged-in buyer
PATCH /api/buyer/all BUYER session { "sellerId": 5 } Adds a farm to the buyer's saved list
DELETE /api/buyer/all/{sellerId} BUYER session — Removes a farm from the buyer's saved list

All three endpoints are already covered by the existing security rule in SecurityConfig.java: /api/buyer/** requires the BUYER role. No security config changes are needed.
